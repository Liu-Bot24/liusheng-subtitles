# 浮光译影

浮光译影是一个网页视频预加载字幕插件。插件会在浏览器里嗅探网页视频媒体源，用浏览器内 Web FFmpeg 抽取适合语音识别的临时音频，再调用用户配置的 ASR 和翻译接口生成字幕。

当前目标是独立 Chrome 插件形态：用户安装插件后即可使用，不需要额外安装或启动配套程序。

## 本地安装

1. 打开 `chrome://extensions`
2. 开启开发者模式
3. 点击“加载已解压的扩展程序”
4. 选择本仓库的 `extension/` 目录

## 使用方式

1. 打开带有网页视频的页面。
2. 打开浮光译影侧边栏。
3. 在“设置”里配置 ASR 和翻译模型档案。
4. 在“任务与字幕”里选择媒体源；必要时指定原音频语言，再开始预加载。
5. 生成后的字幕会显示在侧边栏，并可开启页面浮层字幕。
6. 真实长视频识别异常时，可在字幕区域导出 ASR 诊断归档；归档内包含 `diagnostics.json` 和可离线对照的音频 chunk，用于核对 VAD 区间、原始 ASR 返回和后处理结果。

ASR 和翻译接口可以是公网服务，也可以是局域网内的 OpenAI 兼容接口。插件只按用户填写的 URL 发起请求，不区分服务部署在公网还是局域网。

原音频语言默认自动识别。用户在任务入口指定语言时，插件会把规范化后的语言码作为 ASR 请求的 `language` 字段发送给用户配置的 ASR 接口，用于减少自动语言检测误判。

ASR 档案包含 VAD 过滤策略：默认“自动”。标准 OpenAI、Groq、xAI 请求不会被附加非标准参数；自建 OpenAI-compatible 接口只会在 `openapi.json` 确认支持后自动开启兼容 VAD 字段，并使用 `threshold=0.5`、`max_speech_duration_s=30`、`min_silence_duration_ms=160`、`speech_pad_ms=400` 这一组 Speaches/faster-whisper 成熟参数。若兼容端点同时暴露 Speaches 风格的 `/audio/speech/timestamps`，插件会先请求可靠语音区间，再决定是否跳过无语音分段并作为后处理证据。浏览器内 FFmpeg 记录的 `silencedetect` 区间只作为弱提示，不会单独跳过 ASR 或删除字幕。

对本地或局域网 OpenAI-compatible ASR，插件会读取同源 `openapi.json`。如果接口明确声明支持 Whisper/faster-whisper 常见抑制参数，插件会自动附加 `condition_on_previous_text=false`、`no_speech_threshold` 等兼容字段，用于减少长视频静音、音乐或片尾段幻觉。标准 OpenAI、Groq、xAI 请求不会附加这些非标准字段。

## 当前边界

- 媒体源发现对齐 cat-catch 的浏览器嗅探思路：通过网络请求、页面媒体元素、播放器配置对象和已加载资源发现 m3u8、mpd、m4s、mp4 或音频源。
- 音频处理只使用浏览器内 Web FFmpeg。插件不会自动运行本机 ffmpeg、yt-dlp 或后台守护进程。
- 语音识别和翻译通过用户配置的 API 接口完成，API 密钥只保存在本机浏览器的扩展本地存储中。
- 插件不内置、不启动、也不管理任何 ASR 或翻译服务端。
- 字幕缓存保存在本机浏览器的扩展 IndexedDB 中；下次打开同一链接时会优先加载本地字幕，避免重复消耗 ASR 和翻译额度。
- 需要登录态的媒体请求只依赖浏览器当前 Cookie 和允许的跨域请求能力；插件不提供手填 Cookie 入口，也不在脚本中写入 Cookie 请求头。
- 翻译模型如果返回无法解析、空结果、条数不完整或疑似内容审核错误，插件会保留已成功译文并只补译漏项；整批请求仍失败时才对同一批次逐级二分，最多降到单句，默认 60 条批次最坏内容拆分调用预算不超过 120 次；如果兼容端点首次明确拒绝 `response_format`，最多增加 1 次兼容性探测；无效 API key、额度、限流等账户或配置错误不会切片放大调用。
- ASR 后处理会结合语音区间、ASR 质量信号、word timestamps、重复模式和跨语言片尾/订阅类高频幻觉短语过滤明显错误字幕；短语命中本身不会被无条件删除。源码参考和实现边界见 `implementation notes`。

## 客观限制

- 浏览器插件不能自动安装或启动本机原生命令行工具。
- 浏览器内 Web FFmpeg 受浏览器内存、跨域访问、站点防盗链和 WASM 性能限制。
- DRM、加密签名、需要登录态但禁止跨域读取的媒体源，可能无法被浏览器内处理链路完整抽取。
- YouTube 等平台的真实媒体解析可能需要平台专用签名逻辑；当前不把 yt-dlp 打包进插件。
- 插件不做系统级音频捕获，也不做进程级音频捕获。

## 隐私

插件的隐私说明见 `docs/PRIVACY.md`。字幕缓存和 API 密钥保存在本机浏览器；第三方模型请求只发往用户配置的 ASR 或翻译服务。
