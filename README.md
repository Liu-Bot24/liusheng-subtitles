# 浮光译影

浮光译影是一个网页视频预加载字幕插件。插件会在浏览器里嗅探网页视频媒体源，用浏览器内 Web FFmpeg 抽取适合语音识别的临时音频，再调用用户配置的 ASR 和翻译接口生成字幕。

当前目标是独立 Chrome 插件形态：用户安装插件后即可使用，不需要额外安装或启动配套程序。

旧的 PyQt6 桌面版保留在 `desktop-pyqt6` 分支。

## 项目结构

```text
extension/   Chrome Manifest V3 插件
docs/        架构、隐私和上架说明
web-ffmpeg/  可独立托管的 Web FFmpeg 页面
```

## 开发运行

1. 打开 `chrome://extensions`
2. 开启开发者模式
3. 点击“加载已解压的扩展程序”
4. 选择本仓库的 `extension/` 目录

## 使用方式

1. 打开带有网页视频的页面。
2. 打开浮光译影侧边栏。
3. 在“设置”里配置 ASR 和翻译模型档案。
4. 在“任务与字幕”里选择媒体源并开始预加载。
5. 生成后的字幕会显示在侧边栏，并可开启页面浮层字幕。

ASR 和翻译接口可以是公网服务，也可以是局域网内的 OpenAI 兼容接口。插件只按用户填写的 URL 发起请求，不区分服务部署在公网还是局域网。

## 当前边界

- 媒体源发现对齐 cat-catch 的浏览器嗅探思路：通过网络请求、页面媒体元素、播放器配置对象和已加载资源发现 m3u8、mpd、m4s、mp4 或音频源。
- 音频处理只使用浏览器内 Web FFmpeg。插件不会自动运行本机 ffmpeg、yt-dlp 或后台守护进程。
- 语音识别和翻译通过用户配置的 API 接口完成，API 密钥只保存在本机浏览器的扩展本地存储中。
- 字幕缓存保存在本机浏览器的扩展 IndexedDB 中；下次打开同一链接时会优先加载本地字幕，避免重复消耗 ASR 和翻译额度。

## 客观限制

- 浏览器插件不能自动安装或启动本机原生命令行工具。
- 浏览器内 Web FFmpeg 受浏览器内存、跨域访问、站点防盗链和 WASM 性能限制。
- DRM、加密签名、需要登录态但禁止跨域读取的媒体源，可能无法被浏览器内处理链路完整抽取。
- YouTube 等平台的真实媒体解析可能需要平台专用签名逻辑；当前不把 yt-dlp 打包进插件。
- 插件不做系统级音频捕获，也不做进程级音频捕获。

## 上架准备

插件的隐私说明见 `docs/PRIVACY.md`。提交 Chrome Web Store 时，应说明：插件用于网页视频字幕翻译；全站 host 权限用于在网页加载早期发现媒体源；字幕缓存和 API 密钥保存在本机浏览器；第三方模型请求只发往用户配置的 ASR 或翻译服务。
